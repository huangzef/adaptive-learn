# Milestone Skip Schema

此格式用于记录用户跳过里程碑的原因，供后续会话优化教学策略。

## 存储位置

会话文件的 `## 学习画像` 或 `## 教学复盘` 分区。

## 触发时机

当用户选择跳过某个里程碑时（即用户回复选项 C"跳过"），在执行跳过前先询问跳过原因。

## 字段定义

```yaml
milestone_skip:
  milestone_index: 1
  milestone_name: "里程碑名称"
  reason:
    value: ""      # 实际选中的原因
    options:       # 可选原因列表
      - too_easy: "太简单，已经掌握"
      - not_relevant: "不太相关，不是当前重点"
      - time_pressure: "时间有限，先跳过"
      - other: "其他原因"
  recorded_at: ""  # ISO 8601 时间戳
```

## 快速选项按钮格式

当用户说"跳过"时，提示选择原因：

```
为什么想跳过？
[A] 太简单，已经掌握
[B] 不太相关，不是当前重点
[C] 时间有限，先跳过
[D] 其他原因
```

## 记录到会话文件

跳过原因记录到 `## 学习画像` 分区：

```markdown
### 里程碑跳过记录

- 里程碑 1：[名称] — 跳过原因：太简单，已经掌握 | 2026-03-24
```

同时追加到 `## 教学复盘` 分区：

```markdown
### 里程碑跳过

- 里程碑 1：[名称] — 原因：太简单 | 影响：可提升后续里程碑难度
```

## 使用场景

- **难度调整**：如果某里程碑被多人标记为"太简单"，下次自动减少类似里程碑数量
- **路径优化**：如果某里程碑被多人标记为"不相关"，反思路径设计是否合理
- **节奏提示**：如果频繁出现"时间有限"，下次控制里程碑数量或深度
- **用户意图理解**：累计数据可推断用户的学习风格（快速型 vs 深度型）

## 示例

```yaml
milestone_skip:
  milestone_index: 3
  milestone_name: "异常处理设计"
  reason:
    value: "too_easy"
    display: "太简单，已经掌握"
  recorded_at: "2026-03-24T14:30:00+08:00"
```

会话文件中的记录：

```markdown
### 里程碑跳过记录

- 里程碑 3：异常处理设计 — 跳过原因：太简单，已经掌握 | 2026-03-24
```

---

**Usage note**: 此格式由 mentor-guide 在用户选择跳过里程碑时触发，按钮式选项避免打断学习节奏。learning-session-manager 在会话开始时读取此数据，动态调整教学难度和里程碑数量。
