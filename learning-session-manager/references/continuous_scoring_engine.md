# 连续评分引擎 (Continuous Scoring Engine)

跨技能共享的参数计算模块。所有评分和参数推导必须通过本模块完成。

## 综合能力指数计算 (Composite Ability Index)

综合能力指数是所有参数推导的核心输入，由四个维度加权计算得出。

**公式:**
```
综合能力指数 = 基础能力×0.4 + 目标清晰度×0.2 + 时间资源×0.25 + 学习动机×0.15
```

**验证规则:**
```
min(max(value, 0), 100)
```

最终结果始终钳制在 [0, 100] 范围内。

**维度权重表:**

| 维度 | 权重 | 范围 |
|------|------|------|
| 基础能力 | 0.40 | 0-100 |
| 目标清晰度 | 0.20 | 0-100 |
| 时间资源 | 0.25 | 0-100 |
| 学习动机 | 0.15 | 0-100 |
| **合计** | **1.00** | **0-100** |

## 参数推导公式

### 1. 颗粒度 (Granularity)

单个学习单元的时长，用于划分学习步骤和里程碑。

**公式:**
```
颗粒度 = 15 + 基础能力 × 1.05
```

**范围:** 15-120 分钟

| 基础能力 | 颗粒度 (分钟) | 说明 |
|----------|---------------|------|
| 0 | 15 | 极细粒度，高度引导式 |
| 25 | 41 | 细粒度，较多指导 |
| 50 | 68 | 中等粒度，平衡式 |
| 75 | 94 | 粗粒度，自主性较强 |
| 100 | 120 | 极粗粒度，完全独立 |

---

### 2. 练习密度 (Practice Density)

学习过程中实际练习占总时间的比例。

**公式:**
```
练习密度 = max(30, min(80, 30 + 基础能力 × 0.5))
```

**范围:** 30-80%

**说明:**
- `beginners` (基础能力≈0): 练习密度 = 30% (引导式工作，大量示例)
- `advanced` (基础能力≈100): 练习密度 = 80% (独立练习为主)

| 基础能力 | 练习密度 | 说明 |
|----------|----------|------|
| 0 | 30% | 引导式，30%练习 |
| 25 | 43% | 43%练习 |
| 50 | 55% | 平衡式 |
| 75 | 68% | 68%练习 |
| 100 | 80% | 独立练习，80% |

---

### 3. 检查频率 (Check Frequency)

学习过程中需要人工或系统检查的频率。

**公式:**
```
检查频率 = max(1, 1 + 基础能力 / 30)
```

**范围:** 1-5

| 基础能力 | 检查频率 | 说明 |
|----------|----------|------|
| 0 | 1.0 | 每单元1次 |
| 30 | 2.0 | 每单元2次 |
| 60 | 3.0 | 每单元3次 |
| 90 | 4.0 | 每单元4次 |
| 100 | 5.0 | 每单元5次 |

---

### 4. 理论实践比 (Theory-Practice Ratio)

理论学习与实践练习的时间分配比例。

**公式:**
```
理论实践比 = min(60, 30 + 时间资源 / 3)
```

**范围:** 20-60%

| 时间资源 | 理论实践比 | 说明 |
|----------|------------|------|
| 0 | 30% | 最低理论要求 |
| 30 | 40% | 40%理论 |
| 60 | 50% | 平衡式 |
| 90 | 60% | 理论封顶60% |
| 100 | 63% | 钳制到60% |

---

### 5. 里程碑大小 (Milestone Size)

每个里程碑包含的单元数量。

**公式:**
```
里程碑大小 = max(3, 3 + 综合能力 / 25)
```

**范围:** 3-7

| 综合能力 | 里程碑大小 | 说明 |
|----------|------------|------|
| 0 | 3 | 每3单元一个里程碑 |
| 25 | 4 | 每4单元 |
| 50 | 5 | 每5单元 |
| 75 | 6 | 每6单元 |
| 100 | 7 | 每7单元 |

## 调用协议 (Invocation Protocol)

### 触发事件与责任方

评分引擎的计算由以下5个触发事件驱动，每个事件由对应的调用者负责：

| # | 触发事件 | 负责调用者 | 说明 |
|---|----------|-----------|------|
| 1 | **Bootstrap初始评估** | `learning-session-manager` | 会话开始时的初始能力评估 |
| 2 | **里程碑完成** | `mentor-guide` | 学习者完成一个里程碑后更新评分 |
| 3 | **澄清对话后** | `socratic-tutor` | 目标清晰度提升后触发更新 |
| 4 | **会话开始** | `learning-session-manager` | 确认时间资源可用性 |
| 5 | **热力计活跃** | `mentor-guide` / `socratic-tutor` | 学习动机实时调整 |

### 调用流程

```
触发事件发生
    ↓
确定负责调用者
    ↓
获取当前参数值
    ↓
应用对应公式计算
    ↓
验证结果范围
    ↓
更新对应字段
    ↓
记录到评分历史 (delta-write)
    ↓
如到达检查点，由 learning-session-manager 重新计算完整快照
```

## Delta-Write + Snapshot-Refresh 协议

### 核心原则

- **Sub-skills (mentor-guide, socratic-tutor, project-tracker)**: 只能写入增量 (deltas)，不得直接修改完整快照
- **learning-session-manager**: 负责在检查点重新计算完整快照

### Schema 版本要求

```yaml
schema_version: "1.0"  # 必须在 frontmatter 中声明
```

### Delta-Write 格式

```json
{
  "delta": {
    "field": "基础能力",
    "old_value": 50,
    "new_value": 60,
    "change": "+10",
    "reason": "milestone_completion",
    "timestamp": "2026-03-24T10:00:00Z"
  }
}
```

### Sub-skills 操作规则

1. **只能追加**到 `scoring_history` 数组
2. **不能直接修改** `基础能力`, `目标清晰度`, `学习动机`, `时间资源`, `综合能力指数` 字段
3. **只能通过 delta 形式**记录变化
4. **在检查点**由 `learning-session-manager` 读取历史并重新计算完整快照

### Snapshot-Refresh 触发条件

| 检查点类型 | 触发条件 | 执行者 |
|-----------|---------|--------|
| 里程碑检查点 | 完成一个里程碑 | learning-session-manager |
| 会话检查点 | 每次会话开始 | learning-session-manager |
| 澄清检查点 | 目标清晰度提升后 | learning-session-manager |
| 热力检查点 | 热力计活跃度显著变化 | learning-session-manager |

### 快照重新计算流程

```
1. 读取 scoring_history 中的所有 delta 记录
2. 按时间顺序重放所有变更
3. 计算当前的完整参数值
4. 更新所有派生参数 (综合能力指数等)
5. 生成新的快照并存储
6. 保留历史 delta 记录供审计
```

## Bootstrap 初始评估 (Initial Assessment)

会话开始时，使用以下4个问题进行初始能力评估。

### 问题1: 先验知识评估

**问题:** "请描述你对本主题已有的了解程度"

**映射规则:**

| 回答等级 | 基础能力赋值 |
|----------|-------------|
| L1: 完全不了解 | 0-15 |
| L2: 听说过但不清楚 | 16-35 |
| L3: 理解基本概念 | 36-60 |
| L4: 能独立完成基础任务 | 61-80 |
| L5: 能够教别人 | 81-100 |

### 问题2: 目标清晰度评估

**问题:** "你希望通过本次学习达到什么具体目标？"

**映射规则:**

| 回答等级 | 目标清晰度赋值 |
|----------|--------------|
| L1: 没有具体目标 | 0-15 |
| L2: 有模糊想法 | 16-35 |
| L3: 有明确方向但不确定步骤 | 36-60 |
| L4: 有清晰目标和大致步骤 | 61-80 |
| L5: 目标非常清晰且有详细计划 | 81-100 |

### 问题3: 时间资源评估

**问题:** "你每周能投入多少时间学习本主题？"

**映射规则:**

| 回答等级 | 时间资源赋值 |
|----------|-------------|
| L1: 少于1小时/周 | 0-15 |
| L2: 1-3小时/周 | 16-35 |
| L3: 3-5小时/周 | 36-60 |
| L4: 5-10小时/周 | 61-80 |
| L5: 10小时以上/周 | 81-100 |

### 问题4: 学习动机评估

**问题:** "你学习本主题的主要动力是什么？"

**映射规则:**

| 回答等级 | 学习动机赋值 |
|----------|-------------|
| L1: 外部压力/被迫 | 0-15 |
| L2: 功利性目的 | 16-35 |
| L3: 兴趣驱动 | 36-60 |
| L4: 强烈热情 | 61-80 |
| L5: 使命感/核心驱动力 | 81-100 |

### 初始评估汇总

```
1. 计算各维度加权综合能力指数
2. 根据基础能力确定初始颗粒度
3. 根据基础能力确定初始练习密度
4. 根据基础能力确定初始检查频率
5. 根据时间资源确定初始理论实践比
6. 根据综合能力确定初始里程碑大小
7. 存储初始评估快照
```

## 动态更新规则 (Dynamic Update Rules)

### 基础能力 (Base Ability)

**更新触发:** 里程碑完成

**更新幅度:**
```
基础能力变化 = +5 ~ +15 (基于里程碑表现)
```

**表现评级与调整:**

| 里程碑表现 | 基础能力变化 |
|-----------|-------------|
| 低于预期 | +5 |
| 符合预期 | +10 |
| 高于预期 | +15 |

**计算公式:**
```
新基础能力 = min(100, 旧基础能力 + 变化值)
```

---

### 目标清晰度 (Goal Clarity)

**更新触发:** 澄清对话后 (由 socratic-tutor 触发)

**更新幅度:**
```
目标清晰度变化 = +5 ~ +20
```

**澄清效果评级:**

| 澄清深度 | 目标清晰度变化 |
|---------|--------------|
| 初步澄清 | +5 |
| 部分澄清 | +10 |
| 完全澄清 | +15 |
| 深化目标 | +20 |

---

### 学习动机 (Learning Motivation)

**更新触发:** 热力计 (Heat Meter) 活跃度变化

**更新规则:** 实时调整，基于热力计读数

```
if 热力计读数 >= 80:
    学习动机 += 5 (最高100)
elif 热力计读数 <= 30:
    学习动机 -= 10 (最低0)
else:
    学习动机不变
```

**热力计等级映射:**

| 热力计读数 | 学习动机调整 |
|-----------|-------------|
| 0-30 (低) | -10 |
| 31-50 (中低) | 0 |
| 51-70 (中高) | +3 |
| 71-85 (高) | +5 |
| 86-100 (极高) | +10 |

---

### 时间资源 (Time Resources)

**更新触发:** 每次会话开始确认

**更新规则:**
```
时间资源 = 确认的每周可用小时数 × 10
```

**确认机制:**
- 会话开始时询问: "本周可用于学习的时间是否有变化？"
- 学习者确认后更新
- 默认保持上次值

## 完整参数计算示例

### 示例输入

| 参数 | 值 |
|------|-----|
| 基础能力 | 55 |
| 目标清晰度 | 65 |
| 时间资源 | 50 |
| 学习动机 | 70 |

### 计算过程

**1. 综合能力指数:**
```
综合能力 = 55×0.4 + 65×0.2 + 50×0.25 + 70×0.15
         = 22 + 13 + 12.5 + 10.5
         = 58
```

**2. 颗粒度:**
```
颗粒度 = 15 + 55 × 1.05 = 72.75 → 73分钟
```

**3. 练习密度:**
```
练习密度 = max(30, min(80, 30 + 55×0.5)) = max(30, min(80, 57.5)) = 57.5%
```

**4. 检查频率:**
```
检查频率 = max(1, 1 + 55/30) = max(1, 2.83) = 2.83 → 3
```

**5. 理论实践比:**
```
理论实践比 = min(60, 30 + 50/3) = min(60, 46.67) = 46.67%
```

**6. 里程碑大小:**
```
里程碑大小 = max(3, 3 + 58/25) = max(3, 5.32) = 5.32 → 5
```

### 示例输出

| 派生参数 | 值 |
|---------|-----|
| 综合能力指数 | 58 |
| 颗粒度 | 73 分钟 |
| 练习密度 | 58% |
| 检查频率 | 3 |
| 理论实践比 | 47% |
| 里程碑大小 | 5 |
